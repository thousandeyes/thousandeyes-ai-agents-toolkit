# Contributing

Thanks for your interest in contributing to the ThousandEyes AI Agents Toolkit!
There are many ways to get involved, from reporting issues to submitting pull
requests. This document describes how to participate.

- [Getting Started](#getting-started)
- [How to Report a Bug](#how-to-report-a-bug)
- [Submitting a Contribution](#submitting-a-contribution)
- [Code of Conduct](#code-of-conduct)

## Getting Started

Follow GitHub's official guide for contributing to a project from a fork:
[Contributing to a project](https://docs.github.com/en/get-started/exploring-projects-on-github/contributing-to-a-project).

## How to Report a Bug

If you run into problems using the toolkit, please contact
[ThousandEyes Support](https://docs.thousandeyes.com/product-documentation/getting-started/getting-support-from-thousandeyes#contacting-support).

If you discover a security vulnerability, please **do not** report it through
GitHub. Instead, follow the responsible disclosure process described in
[SECURITY.md](/SECURITY.md).

## Submitting a Contribution

Before submitting a pull request, please make sure:

- Your changes are focused and address a single concern.
- You have tested your changes locally.
- Commit messages are clear and descriptive.
- The pull request description explains the motivation and scope of the change.

All contributions are subject to review by the project maintainers. We aim to
respond to pull requests within a reasonable time frame.

## Code of Conduct

This project adheres to the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md).
By participating, you are expected to uphold this code. Please report
unacceptable behavior to `oss-security@cisco.com`.
